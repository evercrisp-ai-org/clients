# Performance Data Log

> This is the template for capturing real performance data from published content. Copy this file to `outputs/performance-logs/log-[YYYY-MM-DD].md` and fill it in with actual metrics from your platform analytics before running a Recursive Learning cycle (content-recipe.md Section 11, START_HERE.md Session 5).

**Review period:** [Start date] to [End date]
**Content batch reviewed:** [e.g., content-batch-2026-02-12, Weeks 1-2]
**Date logged:** [Date you pulled this data]

---

## Instructions

1. Log in to each platform's analytics dashboard (LinkedIn, YouTube Studio, Facebook Insights, your blog/website analytics).
2. For each piece of published content in the review period, fill in one entry below.
3. Capture data after each piece has been live for at least 7 days (ideally 14) so metrics have time to stabilize.
4. If a metric is not available on a given platform, write "N/A". Do not guess or estimate.
5. After filling in the per-piece data, complete the Batch Summary section at the bottom.
6. Save the file and reference it when running the Recursive Learning cycle.
